## 二、激活码系统（自动发码 + 本地离线校验）

### 流程

```
老板到期点击"续费"
  → App内置浏览器跳转面包多收款页
  → 微信/支付宝扫码付 99 元
  → 面包多回调 → Hermes 自动生成激活码 → 飞书私信推送
  → 老板复制激活码 → 粘贴到 App → 本地 HMAC 离线校验
  → 解锁 365 天
```

### 技术方案

- **激活码格式：** `HMAC-SHA256(机器识别码, 服务器私钥)` → Base64 编码
- **App 本地校验：** App 内置公钥，离线验证签名 + 有效期
- **防破解：** 初期（<100用户）不设服务器校验。盗版出现 = 用户量起来了 = 花200/月租云函数值得
- **成本：** 0 元（除可选域名 35 元/年）

## 三、首次使用引导（替代冷冰冰的6个按钮）

### 问题
现在 App 打开第一屏就是 6 个按钮，老板不知道点哪个。

### 方案
App 启动后，交互面板自动弹出第一条消息：

> 您好！我是您的经营助理，能帮您做几件事：
> 
> 🔔 **每天早上提醒**您哪些客户该催款了
> 📊 **月底自动对账**，帮您找出银行流水和舟谱差的钱
> 📋 算货损、出销售简报
> 
> 我先帮您配好 DeepSeek 密钥，大概 2 分钟。请用手机打开 deepseek.com，注册后点左侧🔑图标创建 API Key，复制后粘贴到下方输入框。

### 配置流程（全部在 App 内完成）
1. 引导配 API Key（内置截图 + 大白话指引）
2. 引导配飞书（内置 App ID / Secret 输入 + 测试连通性）
3. 配完自动说："全部就绪！明天早上开始您就会收到催收提醒。想先试试什么功能？"

## 四、核心功能优化

### 4.1 催收看板（核心卖点，优先级最高）

| 现状 | 优化后 |
|------|--------|
| 只列客户名+金额+天数 | 加**紧急度分级**：🔴超60天可能坏账 / 🟠超30天 / 🟡即将到期 |
| 单次催收提醒 | 加**历史对比**："该客户连续3个月回款延迟，上月超了18天" |
| 干巴巴催款话术 | **个性化话术**：附具体发货单号、日期、金额明细，老板复制即发 |

### 4.2 对账报告（生命线）

| 现状 | 优化后 |
|------|--------|
| 列差异：邮储有舟谱无 +368.50 | 每条差异加**操作指引**：→ "建议在舟谱补录XX客户应收款" |
| 老板看完要自己琢磨怎么办 | 看完照做就行 |

### 4.3 销售简报（从好看→有用）

| 现状 | 优化后 |
|------|--------|
| 报销售额、环比、品类排名 | 改报**回款率分析**："今日回款率62%，环比降8%。主因：XX超市3万逾期" |
| 舟谱也能看到的信息 | 舟谱没有、老板真需要的信息 |

## 五、产品形态

| 类型 | 决策 |
|------|------|
| 桌面 App | ✅ 保留（产品感=值钱感） |
| 飞书推送 | ✅ 保留（消息输出主力） |
| 纯飞书版（无App） | ❌ 不做（显得不专业，数据在别人电脑不放心） |
| 服务器托管 | ❌ 不做（老板不放心数据在外） |

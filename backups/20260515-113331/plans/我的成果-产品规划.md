# 「我的成果」产品规划 v2

> 基于 2026-05-10 讨论整理。核心目标：从"工具"升级为"数字分身"——不只帮干活，还帮实现梦想。成果沉淀在 App 里形成不可替代的粘性。

---

## 战略逻辑

```
极简安装（0门槛）
  → 上手即用（满足需求）
    → 发现"还能这样"（启发需求：PPT/邮件/报表/写书/做App）
      → 成果沉淀在「我的成果」里（情感绑定 + 仪式感）
        → 离不开了（粘性）
          → 高频使用（利润）
```

## 分期规划

### 第一期：基础改造（1-2 天）

**目标：** 「报告」→「我的成果」，搭好框架，现有功能不受影响。

| 事项 | 说明 |
|------|------|
| 侧边栏改名 | 「报告」→「我的成果」 |
| 三分类结构 | 📊 业务报表 / 📖 我的创作 / 🛠 我的工具 |
| 现有报告迁移 | 对账报告、财务报表等 → 📊 业务报表 |
| 成果卡片化 | 每个成果有名称+日期+预览缩略图，不是纯文件列表 |
| 空状态引导 | 每个分类空了放一句引导语（如「还没写书？跟我说"帮我写本书"」） |
| 文件夹结构 | `~/.hermes/成果/业务报表/` `~/.hermes/成果/我的创作/` `~/.hermes/成果/我的工具/` |

**改完后的侧边栏：**
```
功能
  ✍️ 帮我写
  🔍 帮我找
  📅 提醒我
  能干的事

💬 直接聊天

页面
  🏠 主页
  ⏰ 定时任务
  📱 连接手机
  📁 我的成果  ← 升级
```

---

### 第二期：白领场景引导（1-2 天）

**目标：** 新用户打开 App 不只看到空对话框，而是被具体场景启发——"原来还能做这个"。

**改造点：**
1. 主页空状态不再显示「👋 点左边功能按钮开始」，改为滚动式场景卡片
2. 每条卡片是具体的一句话，用户看了就能照着说：

> - 「把这周销售数据做成 PPT，每页一个品牌」
> - 「帮我回今天的工作邮件，紧急的先回」
> - 「用这份 Excel 出利润分析报表，同比环比都算上」
> - 「整理学生成绩单，算出平均分和排名」
> - 「把会议记录整理成待办清单，@对应负责人」
> - 「我口述几个要点，帮我做成汇报 PPT 大纲」

3. 老用户可关闭，不打扰
4. 和「帮我写」按钮互补——按钮是入口，场景卡片是启发

**文案原则：** 不说功能，说场景。不说「支持 PPT 导出」，说「把这周数据做成 PPT」。

---

### 第三期：第一个创作模块——「我写的书」（3-5 天）

**目标：** 让用户 30 分钟内从零产出一章能看的小说/文章/课程。

**创作流程：**
1. 用户在聊天框说「帮我写本书」或点快捷入口
2. Hermes 引导：书名 → 类型/风格 → 大纲 → 第一章
3. 每写一章沉淀到 `📖 我的创作/书名/`
4. 可随时继续写、修改、回退

**成果展示（在「我的成果」里）：**
- 卡片封面：书名 + 自动生成的封面 + 已写章节数
- 点进去：章节目录 + 预览阅读 + 历史版本

**必须打磨：**
- 第一章 30 分钟内产出
- 风格可选：「像金庸」「像刘慈欣」「像我自己写的那样」
- 版本管理：每次修改自动存档，能回退到「上周三那版」（不叫 commit/diff）

**导出：** PDF / ePub

**区别：** 「帮我写」是今日事——写个周报、回个邮件。 「我写的书」是梦想项目——日积月累的创作。 两个共存，互补。

---

### 第四期：第二个创作模块——「我做的应用」（5-7 天）

**目标：** 用户说出想法，30 分钟内看到能用的网页/工具。

**创作流程：**
1. 用户说「帮我做个客户管理小工具」
2. Hermes 问清需求 → 生成 → 预览 → 修改 → 满意
3. 成果沉淀到 `🛠 我的工具/应用名/`

**成果展示：**
- 卡片封面：应用图标 + 名称 + 创建日期
- 点进去：在线预览 + 版本历史

**技术路线：** 先做 Web 应用（HTML/CSS/JS），零依赖可直接打开

**发布：** 一键部署到公网 → 生成分享链接

---

### 第五期：分享与出口（2-3 天）

| 成果类型 | 出口 |
|---------|------|
| 书 | 导出 PDF / ePub / 纯文本 |
| 应用 | 一键部署公网链接 / 打包下载 |
| 报表 | 导出 Excel / PDF（已有） |

不做社交功能。

---

### 第六期：积分定价

| 操作 | 预估消耗 | 用户感知 |
|------|---------|---------|
| 日常写作（周报/邮件） | ~1-2 积分 | 几毛钱，无感 |
| 做一份 PPT | ~5 积分 | 五毛钱做个 PPT |
| 写一章小说 | ~5 积分 | 五毛钱写一章 |
| 做一个简单网页 | ~15 积分 | 一块五做个工具 |
| 做一个完整应用 | ~80 积分 | 8 块钱做个 App |

---

## 不做的事

- ❌ 不做「我的 GitHub」独立入口（发布时作为一条通道即可）
- ❌ 不做社交功能
- ❌ 不做复杂协作

---

## 成功指标

- 新用户 7 天内至少产出一个创作成果
- 产出成果的用户留存率显著高于纯工具用户
- 创作类积分消耗占比从 0 → 30%+

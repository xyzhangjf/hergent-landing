## 二、商业模式

### 一句话：充值制，不设年费。

**新用户：** 注册即送 **200 积分**，不限时，用完为止。

**充值档位：**

| 金额 | 积分 | 说明 |
|------|------|------|
| 10 元 | 1000 | 最低充值 |
| 30 元 | 3000 | — |
| 50 元 | 5500 | 多送 500，鼓励多充 |

**积分消耗：** DeepSeek 实际 API 成本 × 1.5（含 33% 毛利）。

**一次普通聊天：** ~0.5 积分。10 元 1000 积分 ≈ 能聊 ~2000 次。

**用户不需要配 API Key。** 我们用自己 DeepSeek Key 统一代理。

## 三、技术架构

```
用户电脑                              我们的 VPS
┌─────────────────────┐              ┌──────────────┐
│ Hermes AI.app       │              │ API 代理服务  │
│                     │   校验积分    │              │
│ ├─ 干活引擎 ────── 本地执行 ─────  │ ├─ 积分扣费   │
│ │  (读文件/搜信息/  │              │ ├─ 持 DeepSeek │
│ │   设提醒/跑脚本)   │              │ │   Key       │
│ │                   │   API 调用   │ └──────┬───────┘
│ └─ 对话聊天 ─────────────→  ──────→        │
│                                      ┌──────┴───────┐
│                                      │  DeepSeek    │
│                                      │  API         │
└─────────────────────┘              └──────────────┘
```

**核心原则：干活在用户家，算账在我们家。**

| 在用户电脑跑 | 在 VPS 跑 |
|-------------|----------|
| 读文件、写文件 | 积分校验 |
| 联网搜索 | 持有 DeepSeek Key |
| 定时任务 | API 转发代理 |
| Hermes 全部技能 | 用户余额管理 |


## 八、安全设计

**DeepSeek Key 不出服务器。** App 只做两件事：本地干活 + 对话请求转发 VPS。Key 永远不在用户电脑上。

| 风险 | 对策 |
|------|------|
| 用户破解桌面 App | 破解了也只能免费用本地能力，LLM 调用必须走 VPS 积分校验 |
| VPS Key 被盗 | Key 只在服务端环境变量里，不落盘、不打日志 |
| 积分伪造 | 所有 API 调用在 VPS 实时扣费，App 端无法绕过 |

## 十二、不变与不做

**不变：**
- 磨砂玻璃 UI
- 对话面板 + 报告中心
- 定时任务、在线升级

**不做（v1.0）：**
- 不做 Windows 版（验证通过再说）
- 不做移动端 App
- 不做多语言
- 不做技能市场/插件商店
- 不纠结 UI 微调（壳已经够好）

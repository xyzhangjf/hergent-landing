# 舟谱银行流水对账 Skill

## 核心目标

**搞清楚每一笔钱到底到没到、对不对得上。**

输出三种差异的问题清单：
1. ❌ **银行有，舟谱没有** → 可能漏记
2. ❌ **舟谱有，银行没有** → 钱没收到 ⚠️重点关注
3. ⚠️ **金额不一致** → 差多少，为什么差

---

## 业务流程

```
舟谱导出：数据报表 → 账户收支明细 → 选择账户+时间段 → 查询 → 导出Excel
银行导出：从各银行网银/手机银行导出流水Excel
↓ 两者对账 ↓
生成问题清单 → 逐条核查 → 确认差异原因
```

---

## 脚本

### 主对账脚本

```bash
python scripts/reconcile.py \
  --bank <银行流水.xlsx> \
  --finance <舟谱账户收支明细.xlsx> \
  --month "2026年4月" \
  --output ./output
```

**参数：**
| 参数 | 必填 | 说明 |
|------|------|------|
| `--bank` / `-b` | ✅ | 银行流水Excel路径 |
| `--finance` / `-f` | ✅ | 舟谱账户收支明细Excel路径 |
| `--month` / `-m` | ❌ | 月份标签（用于报告命名），如"2026年4月" |
| `--output` / `-o` | ❌ | 输出目录，默认 `./reconcile_output` |
| `--date-window` / `-d` | ❌ | 日期匹配窗口天数，默认3天 |

**示例：**
```bash
python scripts/reconcile.py \
  --bank ~/Downloads/工行2026年4月流水.xlsx \
  --finance ~/Downloads/账户收支明细表202604.xlsx \
  --month "2026年4月"
```

## 匹配策略

### 阶段1：直接匹配
- 条件：金额完全一致 + 日期差≤3天
- 适用：公对公转账、工资发放等

### 阶段2：T+1+手续费匹配
- 适用：微企付、随行付等第三方支付
- 逻辑：银行到账日 = 舟谱收款日+1天
- 手续费率：0.2791%
- 公式：`银行到账金额 ≈ 舟谱收款金额 × (1 - 0.2791%)`

## 差异处理建议

### 银行有、舟谱没有
1. 检查是否在舟谱里漏登记了
2. 如果是第三方支付，可能汇总时间差
3. 确认后补录舟谱或标记为"已核对无需处理"

### 舟谱有、银行没有
1. **最高优先处理**
2. 联系对方确认是否已付款
3. 确认付款方式（转账、票据、现金？）
4. 确认是否有延迟到账

### 金额不一致
1. 检查是否为手续费差异（第三方支付）
2. 检查是否有部分付款
3. 确认是否有退款或折扣

---

## 注意事项

1. **每账户单独对账**：每个银行账户单独导出、单独对账
2. **按月对账**：建议每月初对上月账
3. **自动字段识别**：脚本会自动识别Excel列名，不需要固定格式
4. **日期留空**：导入模板时单据日期不要填，对账时不影响
5. **备份数据**：对账前建议备份原始文件
